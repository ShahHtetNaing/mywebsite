• Error

    Microsoft 365 will not auto update

  • Date

    2024, March

Info

Microsoft Office installations are set by default to automatically update when new updates are released. However, when viewing the Account section in the backstage, you may notice that updates are either disabled or the option to manage updates is unavailable or hidden. For instance, the figure below shows updates being disabled in the backstage view.

The Challenge

This issue occurs if updates have been manually disabled or are managed through Group Policy settings.

The Solution

Check is ther any other user sign in. If there is Sign out the user account and follow these steps.
1. Start Registry Editor. To do this, take one of the following actions:
o In Windows 10 or Windows 8, press the Windows key + X, select Run, type regedit in the Open box, and then press Enter.
o In Windows 7 and Windows Vista, select Start, type regedit in the Start Search box, and then press Enter. If you are prompted for an administrator password or for confirmation, type the password, or select Allow.
2. Locate and then select the following registry key:
Note: If this key does not exist in your Registry Editor, see How to add the registry key via policy below.
Microsoft 365 Apps for enterprise, Office LTSC 2021, Office 2019, and Office 2016
HKEY_LOCAL_MACHINE\software\policies\microsoft\office\16.0\common\OfficeUpdate
Office 2013
HKEY_LOCAL_MACHINE\software\Wow6432Node\microsoft\office\15.0\common\OfficeUpdate
3. Examine the following registry values under the \OfficeUpdate key:
o DWORD: EnableAutomaticUpdates
Values:
0 = automatic updates are disabled
1 = automatic updates are enabled
